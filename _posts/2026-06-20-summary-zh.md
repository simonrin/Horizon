## [imec、台积电、ASML 实现 50nm 栅距 2D CMOS 300mm 晶圆集成](https://www.ithome.com/0/966/443.htm) ⭐️ 9.0/10

imec 与台积电、ASML 合作，首次在 300mm 晶圆上成功集成了采用二维过渡金属二硫属化物（TMD）材料的互补 n 型和 p 型晶体管，实现了 50nm 的接触栅极间距（CPP）。该演示采用单次 EUV 光刻定义 28nm 沟道长度，94% 的晶体管功能正常，开关比超过 10 万。 这一里程碑表明，二维材料晶体管能够以与先进硅节点相当的密度集成，为后硅时代逻辑芯片铺平了道路。它解决了在标准 300mm 晶圆平台上同时集成 n 型和 p 型二维晶体管的长期挑战，使二维半导体更接近产业化应用。 n 型晶体管采用二硫化钼（MoS₂）作为沟道材料，p 型采用二硒化钨（WSe₂）或二硫化钨（WS₂）。团队采用了“反向薄膜晶体管”工艺，使用底部接触结构以降低接触电阻，实现了 75nm 的最小有源宽度和接近 2nm 的等效氧化层厚度。50nm CPP 通过标准 0.33 NA EUV 光刻实现，无需 High-NA EUV 或多重曝光。

**背景**: 接触栅极间距（CPP）是晶体管密度的关键指标，定义为从一个晶体管栅极到下一个栅极的距离。二维过渡金属二硫属化物（TMD）如 MoS₂ 是原子级厚度的半导体，相比硅具有更好的栅极控制能力，可实现进一步微缩。此前在 300mm 晶圆上的二维晶体管演示仅限于单一器件类型或较大间距；这项工作首次将互补集成、EUV 光刻和标准 300mm 工艺结合在一起。

## [IETF 发布 RFC 10008：新的 HTTP QUERY 方法](https://www.ithome.com/0/966/439.htm) ⭐️ 8.0/10

IETF 发布了 RFC 10008，将新的 HTTP QUERY 方法定义为标准跟踪规范。该方法允许携带请求体的安全、幂等只读请求，填补了 GET 和 POST 之间的空白。 这为 API 提供了一种标准化方式来处理复杂的只读查询，而无需滥用 POST，提高了 Web 开发的清晰度和安全性。它支持此类请求的缓存和重试，符合 RESTful 原则。 QUERY 方法被定义为安全且幂等，意味着可以自动重试而不会产生副作用。它引入了 Accept-Query 响应头，并支持通过分配的 URI 进行缓存以供后续 GET 请求使用。

rss · ITHome Feed · 6月19日 15:16

**背景**: HTTP 方法如 GET 和 POST 具有不同的语义：GET 是安全且幂等的，但不能携带请求体；而 POST 可以携带请求体，但不保证安全。开发者经常使用 POST 进行复杂查询，这在语义上是不正确的。QUERY 方法通过结合 GET 的安全性和 POST 的携带请求体能力解决了这一问题。
